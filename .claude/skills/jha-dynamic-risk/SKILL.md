# JHA Dynamic Risk — 동적 위험성평가 고도화 워크플로우

## 언제 사용하는가

- 기상·지형을 위험성평가에 결합하는 "동적 업데이트" 기능 설계·구현 시
- 작업중지 룰엔진(풍속·강우·폭염 임계값) 구현 시
- 외부 공공 API(기상청/에어코리아/V-World) 연동 또는 목업 provider 설계 시
- 장비 의존 기능(CCTV 영상AI·위치 지오펜싱·드론·IoT)을 목업으로 대체하고 추후 교체 가능하게 추상화할 때
- 현장 좌표·격자 변환, 지형 재해 레이어(산사태/홍수/지반) 매핑 시

## 설계 원칙 (리서치 기반)

1. **동적 업데이트 우선** — 현장 이동성·기상 변화 때문에 가장 큰 가치가 여기서 나온다. 문서 자동화보다 위치·기상의 실시간 결합이 우선.
2. **외부 위험 = 내부 위험** — 기상·지형을 별도 모듈로 떼지 말고 위험엔진에 직접 결합. 폭염·호우는 "있으면 좋은 기능"이 아니라 사실상 필수.
3. **Human-in-the-loop** — 자동 작업중지 금지. 시스템은 "권고", 최종 작업중지 판단·책임은 현장소장/사업주. 경보→확인→승인→기록 루프.
4. **프라이버시 우선** — 위치·영상은 개인식별보다 사건(이벤트) 중심. 얼굴인식 배제, 최소 수집, 가명키.
5. **provider 추상화** — 실 API/장비가 없는 PoC에서는 목업 provider로 동일 인터페이스 구현, 추후 코드 변경 최소로 실 연동 교체.

## 구현 가능 vs 목업 대체 (PoC 환경 기준)

### 즉시 구현 (실제 로직 — 외부 호출만 목업 데이터)
- **기상 작업중지 룰엔진**: 정량 임계값 결정테이블 → 순수 함수. 실제 동작.
- **날씨×공종 위험 매핑 매트릭스**: 공종별 강풍/강우/폭염/한파/낙뢰/미세먼지 영향.
- **WGS84 ↔ 기상청 LCC 격자(nx,ny) 변환**: 실제 수학 공식 (Re=6371.00877, grid=5.0km, 표준위도 30/60, 기준점 126E/38N).
- **폭염 체감온도 단계 판정**: 31/33/35/38℃ 4단계 + 휴식 의무(33℃ 2시간당 20분).
- **지형 재해 레이어 플래그 결합**: 산사태/홍수/지하매설물/지반 → 위험요인 가중.

### 목업 대체 (장비/유료 — 추상화 후 추후 교체)
- **기상 데이터**: 기상청 단기예보·특보 실 API → `WeatherProvider` 인터페이스, mock impl. (실 API: data.go.kr 인증키)
- **지오코딩/지도**: V-World Geocoder/WMS → mock 좌표. (실 API: 도메인 등록 인증키)
- **CCTV 영상 AI**: YOLO/DeepStream PPE·추락·협착·연기 감지 → mock 이벤트 스트림.
- **위치 지오펜싱**: UWB/BLE RTLS 위험구역 진입 → mock proximity 이벤트.
- **드론 캡처 / IoT 센서 / 생체신호**: 전부 mock + "추후 연동" 배지.

## 단계별 워크플로우

1. **provider 인터페이스 정의** — `WeatherProvider`, `GeoHazardProvider`, `SiteLocationProvider`. 목업 impl + 실 API impl 자리.
2. **격자 변환 유틸** — 현장 주소→좌표(지오코딩 mock)→LCC 격자.
3. **룰엔진** — (b) 기준표 결정테이블. 입력: 공종 + 기상 컨텍스트 → 작업중지/제한/주의 판정 + 근거 법령.
4. **지형 컨텍스트** — 좌표로 재해레이어 공간질의(mock) → 굴착/사면/수변 플래그.
5. **위험엔진 결합** — 기존 hazard 평가 결과에 기상·지형 위험을 합산/가중. 동적 위험지도 생성.
6. **UI** — 동적 위험 패널(기상 카드·작업중지 경보·폭염 휴식·지형 배지), 역할별 노출.
7. **거버넌스** — 권고+승인 워크플로우, 체감온도 기록 보관, 감사 로그.

## 데이터 모델 (요약)

- `WeatherContext`: grid_x/y, temp, apparent_temp_c, wind_ms, gust_ms, rain_mm_1h, snow_cm_1h, pty, warnings[]
- `GeoHazardContext`: landslide_grade(1~5), flood_risk, underground_utilities[], soft_ground, slope_deg
- `WorkStoppageRule`: rule_id, condition, threshold, trade, legal_ref, action(INFO/WARN/STOP/EVAC), source
- `DynamicRiskAssessment`: weather, geo, triggered_rules[], overall_level, recommendations[], human_approval_required
