# JHA Domain Knowledge — 호반그룹 EHS 도메인 표준

## 언제 사용하는가

다음 상황 중 하나라도 해당하면 이 스킬을 적용한다:
- 위험요인 분류(대공종/중공종/세부항목/재해형태)를 다룰 때
- 강도·빈도·위험등급 산정 규칙을 결정할 때
- LLM 응답에 법적 인용을 강제할지 결정할 때
- gold set·평가 루브릭을 작성할 때
- "추락"·"낙하"·"전도" 같은 동의어 구분이 필요할 때
- 중점등록(O/X) 자동 판정 규칙이 필요할 때

## 표준 권위 순서

근거 충돌 시 다음 순서로 결정한다:
1. **산업안전보건법** (2024 개정) + 동법 시행령·시행규칙 — 법적 강제력 1순위
2. **중대재해 처벌 등에 관한 법률** (2026 운영 기준)
3. **위험성평가 지침** (고용노동부 고시 2023-19호) — KRAS 5×5 매트릭스 정의
4. **KOSHA Guide** (G-시리즈, P-시리즈) — 공정안전·작업위험성평가 가이드
5. **KOSHA-MS** 인증 기준
6. **OSHA Job Hazard Analysis** (3071), **ISO 45001** — 보완 참고

데이터셋과 표준이 충돌하면 **삭제하지 않고 병기**한다. 판단 보류 태그(`[도메인 판단 필요]`)로 사용자(안전관리자)에게 위임한다.

## KRAS 5×5 위험도 매트릭스

### 강도(Severity, 1~5)
- **5 (사망)**: 사망 또는 영구 장해
- **4 (중상)**: 1개월 이상 휴업/입원
- **3 (경상)**: 1주~1개월 휴업/통원
- **2 (찰과상)**: 응급조치 후 작업 복귀
- **1 (위험 노출)**: 실제 부상 없음

### 빈도(Likelihood, 1~5)
- **5**: 연간 반복 (>10회/년)
- **4**: 자주 (4~10회/년)
- **3**: 가끔 (1~3회/년)
- **2**: 드물게 (1회/수년)
- **1**: 희박 (1회/수십년)

### 위험등급(Risk Grade)
강도×빈도 곱(1~25) 기반. **확정 임계 (실데이터 4,469행 역산, 99.22% 재현 — SSOT: `_workspace/02_foundation/safety_risk_matrix_spec.md`)**:
- **상(High)**: 곱 ≥ 16 (즉시 작업 중단·근본 대책)
- **중(Medium)**: 10 ≤ 곱 ≤ 15 (조속한 대책 + 임시 통제)
- **하(Low)**: 곱 ≤ 9 (모니터링 + 표준 대책)

> **곱 16(강도 4×빈도 4) 경계셀 특별 규칙**: 데이터에서 상 249건/중 35건으로 갈리는 인간 판단 영역. LLM은 잠정 "상" + human_review 플래그를 부여하고 자동확정을 금지한다. 안전관리자 확정 전 ERP 등록 차단.
> (이전 가설 "상 ≥15/중 8~14"는 2026-06-10 데이터 역산으로 폐기 — 변경 이력 참조)

## 재해형태 표준 분류 (21종)

데이터 분포 기준 카테고리화:
- **추락 계열**: 추락(792), 전도(685), 도괴(19)
- **물체 충돌 계열**: 낙하(731), 충돌(343), 비래(143)
- **신체 협착 계열**: 협착(522), 절단(4), 말림(3), 베임(76), 찔림(21), 골절(3)
- **에너지 계열**: 감전(221), 화재(128), 폭발(25), 질식(43)
- **건강 계열**: 질환(190), 근골격계(124), 직업성 질환(2)
- **구조 붕괴**: 붕괴(178)
- **기타**: 기타(216)

### 동의어 정규화 규칙
- "낙상"·"떨어짐" → **추락**
- "넘어짐"·"미끄러짐" → **전도**
- "감김"·"끼임" → **협착**
- "충격"·"부딪힘" → **충돌**
- "감전사"·"전기 충격" → **감전**

LLM 출력에서 비표준 표기가 발견되면 정규화 후 표시한다.

## 중점등록(O/X) 규칙

**확정 규칙 (실데이터 역산 — SSOT: safety_risk_matrix_spec.md)**: 중점등록 "O"(518건, 11.6%)는 **등급 "상"과 1:1 완전 일치** (O 518건 전부 상, 상 518건 전부 O). 1차 판정은 이 단일 규칙을 따른다.

2차 보강 (법령 매핑 — 등록 페이로드 검증용):
- 산안법 시행규칙 §43(중점 관리 대상 작업) 해당 작업은 등급 무관 법정 인용 필수
- 중대재해처벌법 시행령 별표 1·2 해당 여부는 별도 표시 (중점등록 판정 조건이 아님)

> **용어 주의**: "중점등록"(산안법상 중점 관리 대상)과 "중대재해"(중대재해처벌법상 사망·중상해 발생 사건)는 전혀 다른 법적 개념이다. UI·보고서에서 중점등록을 "중대재해 등록"으로 표기하지 않는다.

LLM이 중점등록 추천 시 적용된 조건을 함께 출력한다 (감사 추적성). 곱16 경계셀의 중점등록은 "O(잠정)" — 안전관리자 확정 전 확정 표기 금지.

## 법적 인용 의무 매트릭스

| 산출물 영역 | 인용 의무 | 권장 인용 형식 |
|------------|----------|--------------|
| 중점등록 작업의 개선대책 | 필수 | "산업안전보건법 시행규칙 §43" |
| 추락(고소작업 ≥ 2m) | 필수 | "산업안전보건기준에 관한 규칙 §43, §44" |
| 감전 | 필수 | "산업안전보건기준에 관한 규칙 §301~§321" |
| 굴착 | 필수 | "산업안전보건기준에 관한 규칙 §338~§347" |
| 일반 위험요인 | 권장 | KOSHA Guide 번호 (예: G-37-2017) |

LLM 응답에 인용이 누락된 의무 영역은 재생성 트리거 (rag-architect 가드레일).

## Gold Set 작성 규약

각 정답 사례는 다음 JSON 스키마:
```json
{
  "id": "GS-0001",
  "task_input": "5층 옥상에서 타워크레인 분해 작업",
  "expected_classification": {
    "major_type": "가설공사",
    "sub_type": "타워크레인(T형)",
    "detail_item": "해체 작업"
  },
  "expected_hazards": [
    {"accident_type": "추락", "must_include": true},
    {"accident_type": "낙하", "must_include": true},
    {"accident_type": "협착", "must_include": false}
  ],
  "expected_grade": {"severity_range": [4,5], "frequency_range": [2,3], "grade": "상"},
  "expected_critical_register": "O",
  "acceptable_variants": {
    "controls_paraphrase_allowed": true,
    "synonym_map_applied": true
  },
  "legal_refs_required": ["산업안전보건법 시행규칙 §43"]
}
```

`must_include: true`는 누락 시 coverage 메트릭 차감. `false`는 포함되면 가산, 누락 무영향.

## 평가 루브릭 (LLM-as-judge 기준)

| 메트릭 | 척도 | 임계치 (PoC 목표) |
|--------|------|----------------|
| Classification accuracy | 0~1 (정확 1.0 / 부분 0.5 / 오 0) | ≥ 0.85 |
| Hazard coverage | recall | ≥ 0.80 |
| Risk grade alignment | 정확 일치 비율 | ≥ 0.75 |
| Citation precision | 인용한 source_row의 실제 근거성 | ≥ 0.90 |
| Citation recall | 누락된 근거 source_row | ≥ 0.70 |
| Faithfulness | LLM-judge 점수 (1~5) | ≥ 4.0 |
| Refuse appropriateness | refuse 발동 정합 | ≥ 0.90 |

## Refuse-to-Answer 정책

LLM은 다음 조건에서 응답을 거절한다:
1. RAG 검색 결과 0건
2. 검색 score 모두 임계치(예: cosine 0.5) 미만 → "유사 사례 없음" 경고만 표시
3. 입력이 작업 설명이 아닌 경우(스팸·테스트·관련 없는 질문)
4. PII 의심 입력(이름·주민번호 패턴 감지)

거절 응답 형식:
> "관련 표준 데이터를 찾지 못했습니다. 안전관리자에게 직접 문의하세요. (사유: {reason})"

## 적용 우선순위 (요약)

1. **법령 위반 가능 케이스는 무조건 인용 의무**
2. **중점등록은 자동 판정 + 적용 조건 함께 표시**
3. **등급 산정은 KRAS 5×5 + 데이터 분포 임계곱 우선**
4. **추측 금지** — 데이터셋·표준에 근거 없는 위험요인은 생성하지 않음
5. **충돌 시 병기 + 사용자 위임**
