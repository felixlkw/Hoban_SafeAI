# RAG Embedding Choice — 임베딩 모델 비교·선정 (Foundation)

> 작성: rag-architect · Phase 2 (Foundation) Wave 2
> 결정: **BGE-M3 선정** (Phase 4 활성화). PoC 베이스라인은 dense 미사용(BM25 단독).
> 목적: 한국어 도메인·온프레미스 보안 제약 하 임베딩 모델 선정 근거 SSOT. 변경 시 재인덱싱 비용 큼 → data-engineer 협업 필수.

---

## 1. 선정 배경

본 시스템 가치는 **"사내 안전 DB와 타 현장 사례로만 답하는 신뢰성"**이며, 외부 LLM 화이트리스트·사내 보안 가이드라인(CLAUDE.md 제약)을 따른다. 따라서 임베딩 모델은:
- 한국어 건설안전 도메인 어휘(타워크레인·흙막이·동바리·항타 등)에 강하고
- **온프레미스(로컬) 구동 가능**(데이터가 외부 API로 나가지 않음)해야 한다.

이 보안 요건이 OpenAI embedding(외부 API)을 사실상 배제하는 핵심 축이다.

## 3. 선정 근거 — BGE-M3

1. **온프레미스 + 라이선스(MIT)** — 안전 DB가 외부로 나가지 않는다. 사내 보안 가이드라인·외부 LLM 화이트리스트 정책과 정합. OpenAI는 이 한 축에서 탈락(데이터 외부 전송).
2. **하이브리드 검색 네이티브** — BGE-M3는 한 모델로 Dense+Sparse(lexical)+ColBERT를 동시 산출한다. 본 시스템의 BM25(sparse)+Dense 하이브리드 설계(`rag_retrieval_spec.md`)와 자연스럽게 결합하며, 향후 Dense·Sparse를 단일 모델로 일관 운영 가능.
3. **장문·복합명사 견고성** — 8192 토큰 지원으로 행 단위 청크(평균 53.9 토큰, 메타 inline 포함)에 여유. 건설 복합명사("시스템 동바리 잭베이스", "COUNTER JIB/FRONT JIB")에 강건.
4. **한국어 다국어 검색 성능** — 다국어 retrieval에서 e5-large와 동급 이상, KoSimCSE(STS 특화로 검색은 약함)보다 검색 적합.
5. **data-engineer 정합** — `etl_pipeline.md §5`에서 dense stub을 BGE-M3로 이미 전제. 추가 모델 선택 분기 없음.

### 차순위·기각 사유
- **multilingual-e5-large**: 견고한 대안(2순위). 단 512 토큰 한계·sparse 미산출로 하이브리드 일관성이 떨어진다. BGE-M3 활성화 실패/성능 미달 시 fallback 후보로 보존.
- **KoSimCSE**: 경량·국산 STS 강점이나 **검색(retrieval)이 아닌 문장유사도 특화** → RAG 검색 recall에 부적합. 재순위(rerank) 단계 보조 후보로만 검토.
- **OpenAI**: 성능은 충분하나 **데이터 외부 API 전송**이 본 PoC 보안 전제와 충돌. 기각.

## 4. 사내 보안 관점 평가 (온프레미스)

| 항목 | BGE-M3 평가 |
|------|------------|
| 데이터 경계 | 임베딩 추론이 사내 인프라 내에서 실행 → 안전 DB 청크가 외부로 나가지 않음 |
| 화이트리스트 정책 | 외부 LLM 화이트리스트와 무관(로컬 모델). LLM(Claude)만 화이트리스트 관리 대상 |

| GPU 요건 | Phase 4 온프레미스 GPU 환경 구비 시 활성화(`etl_pipeline.md §5` skip 사유와 일치) |

## 5. 재인덱싱 비용 경고 (임베딩 변경 시)

임베딩 모델 변경은 **전체 dense 인덱스 재구축**을 동반한다(4,469 청크 전수 재임베딩). 다음을 강제한다:
1. data-engineer에게 **재인덱싱 task 생성 필수**(blue/green alias로 다운타임 0).
2. eval-engineer 회귀 평가 필수(검색 메트릭 비교).
3. 본 문서 변경 이력 갱신 + `rag_retrieval_spec.md` 가중치 재튜닝.

> PoC 베이스라인은 dense 미사용이므로 본 선정은 **Phase 4 활성화 시점에 발효**한다. 그 전까지 BGE-M3는 stub 상태.
