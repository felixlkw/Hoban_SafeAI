# Safety Legal Citation Matrix — 법적 인용 의무 매트릭스 (Foundation)

> 작성: safety-domain-expert · Phase 2 (Foundation)
> 근거: `jha-domain-knowledge` SKILL 인용 매트릭스 + 산업안전보건법/산업안전보건기준에 관한 규칙(이하 "안전보건규칙") + KOSHA Guide + 데이터 빈도 상위 작업
> 권위 순서: 산안법·시행령·시행규칙 > 중대재해처벌법 > 위험성평가 고시 > KOSHA Guide > KOSHA-MS > OSHA/ISO
> 목적: LLM 응답에 **어느 작업·재해형태에 어떤 법조문 인용이 필수/권장인지** 결정. rag-architect 가드레일(누락 시 재생성)·eval citation recall·ERP 등록 게이트의 SSOT.

---

## 0. 인용 의무 레벨 정의

| 레벨 | 의미 | LLM 처리 | eval |
|------|------|---------|------|
| **필수(MUST)** | 누락 시 법적 리스크. 인용 없으면 재생성 트리거 | 응답에 조문 반드시 포함 | 의무영역 recall ≥ 0.95 |
| **권장(SHOULD)** | 포함 시 가산. 누락 무벌점 | 가능하면 KOSHA Guide 등 포함 | recall ≥ 0.70 |
| **금지/주의** | 데이터 갭 영역. 근거 없는 인용 금지 | refuse 또는 경고 | faithfulness 차감 |

> 인용 형식 표준: `"산업안전보건기준에 관한 규칙 §43"`, `"산업안전보건법 시행규칙 §43"`, `"KOSHA Guide C-2"`. 조문 번호는 데이터 근거(source_row)와 함께 표시(citation precision).

## 4. Refuse·인용 충돌 처리 (데이터 갭 영역)

다음 영역은 **인용 의무는 살아있으나 데이터 근거가 부족**하여, LLM이 조문은 표시하되 **구체 대책은 생성하지 말고 경고/refuse**한다(R3 환각 방지 우선).

| 갭 영역 | 인용(표시) | 대책 생성 | 처리 |
|---------|-----------|----------|------|
| 밀폐공간/질식 | §619~§625 표시 | 데이터 빈약 시 생성 금지 | "밀폐공간 절차는 유사 사례 부족 — 안전관리자/KOSHA P-93 확인" 경고 |
| 화학물질/MSDS | 산안법 §110~§115 표시 | **생성 금지** | refuse. scope §1.2 2차 영역 |
| 석면 해체 | 석면안전관리법·산안법 §123 표시 | **생성 금지(full refuse)** | GS-0035. 별도 법령 영역 |
| 작업환경측정 | §125 표시 | 생성 금지 | 데이터 갭 |

> **원칙**: 조문 인용 ≠ 대책 보증. 데이터 근거 없는 영역은 "법적으로는 ○○ 조문 적용 대상이나 본 시스템 데이터에 표준 사례가 부족합니다"로 분리 응답. 인용만으로 환각 대책을 정당화하지 않는다.

## 5. 컴포넌트별 적용 (송신)

| 대상 | 적용 |
|------|------|
| **rag-architect** | §1·§2 필수 영역에서 인용 누락 시 재생성 가드레일. §4 갭 영역은 refuse+조문표시 분기 |
| **eval-engineer** | citation recall: 의무영역(추락·낙하·붕괴·감전·협착·화재·중점등록 O) ≥ 0.95, 전체 ≥ 0.70. precision ≥ 0.90 |
| **backend-engineer** | API 응답 `citations[]` = {law, article, level(MUST/SHOULD), source_row}. 필수 누락 시 응답 검증 실패 |
| **frontend-engineer** | 인용 사이드패널에 MUST는 강조(빨강), SHOULD는 보조. source_row 추적 링크 |
| **erp-integration** | ERP 등록 전 게이트: 중점등록 O·등급 상 레코드는 §43 등 필수 인용 필드 비어있으면 **등록 차단** |
