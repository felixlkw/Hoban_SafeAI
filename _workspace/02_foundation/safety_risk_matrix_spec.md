# Safety Risk Matrix Spec — KRAS 5×5 매트릭스 정합 명세 (Foundation)

> 작성: safety-domain-expert · Phase 2 (Foundation)
> 근거: 데이터 전수 역산(`safety_scope.md §3`, `data_profile.md §7`) + `jha-domain-knowledge` SKILL
> 목적: LLM·RAG·eval·ERP가 공유하는 **등급 산정·중점등록·경계셀 단일 명세(SSOT)**.

---

## 1. KRAS 5×5 강도·빈도 정의 (확정)

SKILL 정의를 그대로 채택(데이터와 충돌 없음).

### 강도(Severity, 1~5)
| 값 | 정의 |
|:--:|------|
| 5 | 사망 또는 영구 장해 |
| 4 | 중상 (1개월 이상 휴업/입원) |
| 3 | 경상 (1주~1개월 휴업/통원) |
| 2 | 찰과상 (응급조치 후 복귀) |
| 1 | 위험 노출 (실제 부상 없음) |

### 빈도(Likelihood, 1~5)
| 값 | 정의 |
|:--:|------|
| 5 | 연간 반복 (>10회/년) |
| 4 | 자주 (4~10회/년) |
| 3 | 가끔 (1~3회/년) |
| 2 | 드물게 (1회/수년) |
| 1 | 희박 (1회/수십년) |

> 데이터 분포(강도 1=581·2=647·3=1,342·4=1,231·5=668 / 빈도 1=672·2=781·3=1,474·4=1,040·5=502). 전부 정수 1~5, 결측·범위위반 0. LLM은 강도·빈도를 **반드시 정수 1~5로 산정**하고 위 정의에 사상한다.

## 2. 등급 임계곱 — 확정 명세 (데이터 역산)

### 2.1 확정 규칙

```
하 (Low)    : 강도 × 빈도 ≤ 9
중 (Medium) : 10 ≤ 강도 × 빈도 ≤ 15
상 (High)   : 강도 × 빈도 ≥ 16
            └ 단, 강도4×빈도4=16 셀은 경계셀 특별 규칙(§3) 적용
```

### 2.2 데이터 역산 근거 (전수 집계, 4,469행)

| 곱(S×F) | 하 | 중 | 상 | 판정 | 결정성 |
|--------:|---:|---:|---:|------|--------|
| 1~9 | 2,444 | 0 | 0 | **하** | 100% 결정적 |
| 10 | 0 | 282 | 0 | 중 | 100% |
| 12 | 0 | 847 | 0 | 중 | 100% |
| 15 | 0 | 343 | 0 | 중 | 100% |
| **16** | 0 | **35** | **249** | **혼합** | **87.7% 상** (경계셀) |
| 20 | 0 | 0 | 261 | 상 | 100% |
| 25 | 0 | 0 | 8 | 상 | 100% |

> 곱 7·11·13·14·17~19·21~24는 5×5 격자에서 산출 불가(부재 정상). **곱16 셀을 제외한 전 셀은 100% 결정적**. 단순 곱 임계 규칙의 전수 일치율 = 4,434/4,469 = **99.22%**. 불일치 35행은 전부 곱16(§4).

### 2.3 5×5 셀별 등급 맵 (확정, * = 경계셀)

```
강도\빈도   f1     f2      f3      f4        f5
s1          하     하      하      하        하
s2          하     하      하      하        중(10)
s3          하     하      하(9)   중(12)    중(15)
s4          하     하      중(12)  상*(16)   상(20)
s5          하     중(10)  중(15)  상(20)    상(25)
```

### 2.4 SKILL 가설과의 차이 — 병기 (충돌 삭제 금지)

| 항목 | 데이터 역산 (확정) | SKILL 기재 가설 | 처리 |
|------|-------------------|----------------|------|
| 하 경계 | 곱 ≤ 9 | 곱 < 8 | 데이터 우선. 곱8·9는 전부 '하' 실측. SKILL은 곱8을 중으로 잘못 추정 |
| 중 경계 | 10 ≤ 곱 ≤ 15 | 8 ≤ 곱 < 15 | 데이터 우선. 곱15는 전부 '중'(상 아님) |
| 상 경계 | 곱 ≥ 16 | 곱 ≥ 15 | 데이터 우선. 곱15 '상' 행 0건 |

> **확정**: PoC 전 컴포넌트는 **데이터 역산값(하≤9 / 중10~15 / 상≥16)**을 사용한다. SKILL 가설값은 부정확하므로 **SKILL 본문 갱신 권고**(jha-domain-knowledge KRAS 섹션). 본 문서가 등급 임계곱의 SSOT.

## 3. 경계셀 특별 규칙 — 강도4×빈도4 (곱16) [도메인 판단 필요]

### 3.1 현상
- 곱16 케이스 **284행 = 상 249 / 중 35** (혼합, 유일한 비결정 셀).
- **결정 패턴: 상 249행은 전부 중점등록 O, 중 35행은 전부 중점등록 X**. 즉 평가자가 작업 맥락(통제 가능성)으로 상↔중을 수동 조정.
- 35행(중 강등) 예시: row 10(타워크레인·낙하), row 416(가교·기타), row 2179(맨홀·전도). 재해형태는 고위험인데 통제 가능성 고려로 중 처리.

### 3.2 LLM 처리 규칙 (확정)

```
IF 강도==4 AND 빈도==4 (곱==16):
    grade = "상"                        # 기본값: 보수적으로 상
    boundary_cell_flag = true           # 경계셀 플래그 부착
    human_review_required = true        # 자동 확정 금지
    message = "경계 셀(강도4×빈도4) — 안전관리자 확인 필요"
    critical_register = "O (잠정)"       # 상 기본이므로 O 잠정, 확정은 사람
ELSE:
    grade = 임계곱 규칙(§2.1) 적용 (결정적)
    boundary_cell_flag = false
```

**원칙**:
1. **자동 확정 금지** — 곱16은 LLM이 단독으로 등급을 확정하지 않는다.
2. **기본 '상' (보수적)** — 누락 시 위험이 큰 방향으로 기본값 설정(고위험 과소평가 리스크 R1 회피).
3. **인간 판단 플래그 필수** — UI/API에 `human_review_required=true` 노출(frontend·backend 송신).
4. **eval 부분점수** — 곱16 케이스는 상·중 **양쪽 정답 허용**(부분점수). rubric §에서 정의.

> 다른 모든 셀(곱16 제외)은 결정적 규칙이므로 자동 확정 가능.

## 4. 모순 35행 목록 및 처리 방침

data_profile §7 + scope §3.5 식별. 전부 곱16(강도4×빈도4)에서 표기='중', 다수결='상'인 행. **표기 오류가 아니라 경계셀 인간 조정 결과**이므로 보정하지 않고 플래그 부여.

### 4.1 확인된 행번호 (data_profile 샘플 12건 실측 검증)

```
Excel row: 10, 416, 419, 427, 559, 1042, 1051, 1382, 1532, 2179, 2436, 3149
(+ 나머지 23행 — data-engineer가 곱16 AND 등급='중' 필터로 전수 추출)
```

전수 추출 쿼리(data-engineer 송신): `강도==4 AND 빈도==4 AND 위험등급=='중'` → 35행.

### 4.2 처리 방침 (확정)

| 방침 | 내용 |
|------|------|
| **원본값 유지** | 35행의 등급='중'·중점등록='X'를 **보정하지 않는다**. 평가자 의도(통제 가능 작업의 중 강등)를 존중. KRAS 매트릭스 위반이 아니라 경계셀 재량. |
| **메타 플래그** | `risk_grade_check=true` + `boundary_cell=true` 부여(data_profile §D11 지지). |
| **eval 처리** | 이 35행을 정답으로 쓰는 gold set 항목은 상·중 양쪽 허용(부분점수). |
| **LLM 학습 신호** | few-shot에 곱16 중 강등 예시를 1~2건 포함하여 "곱16은 맥락 의존"임을 학습. 단 기본값은 상. |

> **충돌 병기**: 순수 곱 규칙이면 35행은 '상'이어야 하나 데이터는 '중'. → 삭제·보정 없이 양쪽 명시 + `[도메인 판단 필요]` 태그 유지. 이 35행은 "데이터와 규칙의 정당한 불일치"이지 오류가 아니다.

## 5. 중점등록(O/X) 규칙

### 5.1 데이터 실측 핵심 발견

```
중점등록 O = 518건, 전부 등급 '상'
등급 '상'  = 518건, 전부 중점등록 O
→ 본 데이터에서 [중점등록 O] ⇔ [등급 '상']  (1:1 완전 일치)
```

곱16 경계셀에서도 일관: 상 249행=O, 중 35행=X. 즉 중점등록은 **등급 산정 결과에 종속**.

### 5.2 확정 규칙 (1차 — 단순규칙)

```
critical_register = "O"  IF  grade == "상"
critical_register = "X"  IF  grade ∈ {중, 하}
(곱16 경계셀: grade 잠정 '상' → critical_register "O (잠정)", human_review_required)
```

### 5.3 SKILL 가설과의 차이 — 병기

| 항목 | 데이터 실측 | SKILL 가설 | 처리 |
|------|------------|-----------|------|
| O 조건 | **단일: 등급=='상'** | 4조건 OR (고위험 OR 법적의무 OR 반복사고 OR 중대재해가능성) | 1차 단순규칙(데이터), 2차 법령 보강 |

> SKILL의 "법적의무 기반 O"(타워크레인 해체 등)는 **본 데이터에 별도로 존재하지 않는다**(법적의무 작업도 등급 상이면 O, 아니면 X). 따라서 PoC 1차는 등급 종속 단일규칙.

### 5.4 2차 보강 방침 (법령 매핑)

PoC 1차 이후, 법정 중점관리 대상 작업(산안법 시행규칙 §43: 타워크레인 설치/해체·굴착·고소작업 등)은 **등급과 무관하게 O 후보로 승급**하는 2차 룰 추가 검토. 단 본 데이터는 이를 지지하지 않으므로(O는 전부 등급 상), 2차 보강은 **추가 법령 데이터 확보 후** 적용. 1차에서는 LLM이 법정 대상 작업에 등급이 중/하로 나오면 `legal_critical_candidate=true` 정보 플래그만 부착(자동 O 승급은 안 함, 안전관리자 검토 유도).

### 5.5 감사 추적성
LLM이 중점등록 O 출력 시 적용 조건 명시: `"중점등록 O (사유: 위험등급 상)"`. 곱16이면 `"O (잠정, 경계셀 확인 필요)"`.
