# Safety Citation Equivalence — 인용 동등행 판정 기준 + gold set v2 확장 근거

> 작성: safety-domain-expert · 근거: `20260612_openai_gpt41_baseline_analysis.md`(E-CITE 13건) + `chunks.jsonl`(dup_group·content_hash) + `safety_taxonomy_review.md §3`(트리플 중복) + `safety_gold_set.jsonl`(v1 35건)
> 산출 연동: `safety_gold_set_v2.jsonl`(비파괴 확장)
> 목적: gpt-4.1 baseline에서 citation_precision 0.336 / recall 0.618의 **근본원인(행-ID 입도 불일치)**을 gold 측에서 해소한다. 환각이 아니라, 의미상 동등한 다른 행을 인용했을 때 오답 처리되던 문제를 **동등행 집합 채점**으로 교정한다.

## 1. 동등행 판정 기준 (보수적·4조건 AND)

한 정답행 r에 대해 후보행 r′가 **다음 4조건을 모두 충족**할 때만 `acceptable_source_rows`에 포함한다. 하나라도 불충족 시 **비동등(오답 유지)**.

| # | 조건 | 운영 정의 | 불충족 예 |
|---|------|----------|----------|
| C1 | **같은 중공종** (`sub_type` 일치) | 대공종만 같은 것은 불충분. 중공종 ID(`sub_type_id`)까지 일치. 단 gold가 의도적으로 다중 sub_type 셋(예: GS-0002 토류벽·SOIL NAIL·SK판넬)을 보유한 경우, 그 셋은 gold가 명시한 것만 인정하고 **자동 교차 확장하지 않는다**. | T형 ↔ L형 타워크레인은 별개 sub_type(단, gold가 이미 쌍으로 보유한 경우는 그대로 유지) |
| C2 | **같은 재해형태** (`accident_type` 일치) | 21종 enum 일치. 추락↔낙하 등 인접형태도 별개로 취급(taxonomy_review §2.1 판정축 적용). | 추락 ↔ 낙하, 협착 ↔ 전도 |
| C3 | **실질 동일 위험 메커니즘** | 동일 작업단계·동일 위험 발생기전을 표현차이만으로 기술. "안전방망 미설치 → 추락"과 "발판/방망 미설치 → 추락"은 동일. "선회브레이크 미작동 붕괴"와 "로드셀 미작동 붕괴"는 **다른 기전 → 비동등**. | 같은 sub+acc라도 발생원인(장비·부재·단계)이 다르면 비동등 |
| C4 | **개선대책 상호 대체 가능** | r′의 대책이 r의 대책을 실질 커버. 한쪽에만 있는 **필수(법정·핵심) 대책이 누락되면 비동등**. 예: 질식에서 "환기장치·가스농도측정·감시인"이 한쪽에만 있으면 비대체. | 681(환기·가스측정 추가) ≠ 675(열풍기 점검만) → 비동등 |

**보수성 원칙**: 위양성(비동등을 동등으로 포함)은 평가를 무의미하게 만든다. 애매하면 **제외**한다. 본 확장은 같은 sub+acc 후보 중 **고유사·대책대체 가능한 소수만** 채택했다(총 +10행).

### 1.1 자동 포함 규칙 (도메인 검토 불요)
- **`dup_group` 동일** (트리플(중공종,세부항목,위험요인) 중복): 위험요인 텍스트 동일이 전제 → C1~C3 자동 충족. 대책 동일 여부만 확인.
- **`content_hash` 동일** (완전중복): 4조건 전부 자동 충족 → 무조건 포함.
- 본 데이터에서 gold 관련 자동대상: DG001[263,265](GS-0029, 이미 gold 보유), DG002·content_hash[335,336](GS-0003, 이미 gold 보유).

### 1.2 비동등으로 유지하는 전형 패턴 (위양성 방지)
- 같은 sub+acc지만 **위험 발생부재/장비/작업단계가 다른** 행 (예: GS-0001 붕괴의 선회브레이크/로드셀/상부불균형/지내력 — 4개 모두 별개 기전).
- **필수 대책이 한쪽에만** 있는 행 (예: GS-0010 681의 환기·가스측정·감시인).
- **등급·강도가 크게 달라 위험 평가맥락이 분기**하는 행 (예: GS-0028 2860 흙막이붕괴 '상' vs 2864 관내토사붕괴 '중').
- gold가 **의도적으로 분기쌍**(낙하/근골격계, 추락/협착 등)으로 설계한 케이스의 다른 슬롯.

### 2.1 E-CITE 판정 요약
- **동등 인정(부분 확장) : 8건** — GS-0003, 0004, 0005, 0006, 0008, 0010, 0017 + GS-0029(이미 완비).
  - 단 GS-0029는 신규 추가 0행(기존 완비)이므로 **"신규 확장으로 동등 인정"은 7건**, GS-0029 포함 시 **8건이 동등성 반영 상태**.
- **오답 유지 : 5건** — GS-0001, 0002, 0009, 0022, 0026. (동등 후보 부재 또는 LLM cited가 진짜 비동등)
- **핵심 구분**: GS-0001·0002의 LLM cited 행([42,43,44] / [1723,2126,...])은 **진짜 비동등**이므로 확장하지 않고 오답 유지 — 이 보수성이 평가 유효성을 지킨다.

## 4. eval-engineer 송신 — 러너 채점 로직 변경 가이드 (3줄)

1. **필드 교체**: 인용 채점 시 `expected_source_rows`(v1) 대신 **`acceptable_source_rows`(v2 집합)**를 정답집합으로 사용. recall/precision 모두 이 집합과의 교차로 계산(`cited ∩ acceptable`).
2. **precision 재정의**: cited 행이 acceptable 집합에 **하나라도 포함되면 그 인용은 적중**으로 본다(set-membership). precision = |cited ∩ acceptable| / |cited|, recall = (acceptable 중 적어도 1행 적중 여부) 또는 케이스별 임계(≥1행 적중 시 케이스 PASS) — eval_plan의 기존 정의에 맞춰 `metric_citation`만 집합 입력으로 교체.
3. **회귀 baseline 재계산**: 동일 gpt-4.1 응답을 v2 집합으로 재채점하여 citation_precision/recall 재측정 후 본 수치를 신 baseline으로 등록. gold 변경이므로 `dataset_hash` 갱신 필수, 코드·프롬프트 무변경 상태에서 **순수 gold 효과**만 분리 보고할 것.
