## 2. 재해형태 21종 검토 — 동의어/유사 카테고리 구분 기준 명문화

데이터의 21종은 KRAS·KOSHA 재해분류와 정합한다(데이터 분포 검증 완료). 문제는 **분류는 정합하나 의미가 인접해 LLM이 혼동**하는 쌍이 다수라는 점. 아래에 구분 기준을 명문화한다.

### 2.1 구분 기준 명문화 (혼동 위험 쌍)

#### (A) 베임 / 찔림 / 절단 — 날카로운 물체에 의한 상해
| 형태 | 데이터 | 구분 기준 (명문화) |
|------|-------:|-------------------|
| **베임** (76) | 칼날·철판 모서리·그라인더 등에 **피부가 갈라지는** 상해 | 1순위 일반 절상. 그라인더·커터·철근 단부 접촉. |
| **찔림** (21) | 못·철근·드릴 등 **뾰족한 끝이 관통**하는 상해 | 관통·자상. 노출 철근, 못 밟음 포함. |
| **절단** (4) | 신체 일부가 **분리**되는 중대 상해 (손가락 절단 등) | 강도 통상 높음. 절단기·프레스. 베임과 달리 신체 분리. |

> 판정 트리: 신체 분리 → **절단** / 관통·자상 → **찔림** / 그 외 갈라짐 → **베임**. 데이터 표본이 절단 4·찔림 21로 극소 → LLM이 과대 분류하지 않도록 베임을 기본값으로 두되, 입력에 "관통/박힘/꽂힘"·"절단/잘림" 명시어가 있을 때만 찔림·절단 승급.

#### (B) 전도 / 도괴 / 붕괴 — 넘어짐 vs 구조물 무너짐 (★최고 혼동 위험)
| 형태 | 데이터 | 구분 기준 (명문화) |
|------|-------:|-------------------|
| **전도** (685) | **사람 또는 장비가 넘어짐/미끄러짐/뒤집힘** (개인 수준) | 작업자 넘어짐, 굴착기·항타기 전도, 자재 운반 중 넘어짐. 동의어: 넘어짐·미끄러짐·뒤집힘. |
| **도괴** (19) | **가설구조물·적치물이 무너짐** (비교적 소규모·국부) | 비계·적재물·가설재 무너짐. 붕괴보다 국부적. |
| **붕괴** (178) | **지반·토류벽·거푸집/동바리·구조체가 무너짐** (대규모·매몰 위험) | 굴착 사면붕괴, 동바리 붕괴, 토류벽 붕괴. 매몰사망 직결. |

> 판정 트리: 대상이 **사람/장비**가 넘어짐 → **전도** / 대상이 **가설·적치물** 국부 무너짐 → **도괴** / 대상이 **지반·구조체·동바리** 무너짐(매몰) → **붕괴**. 핵심 구분축 = "넘어지는가(전도) vs 무너지는가(도괴/붕괴)" + "국부(도괴) vs 지반·구조(붕괴)".
> 데이터 근거: row 411 "항타기…전도"(장비 넘어짐=전도) vs row 2078 "굴착…안식각 미확보…붕괴"(사면=붕괴). 명확히 분리됨.

#### (C) 추락 / 낙하 / 비래 — 떨어지는 주체
| 형태 | 데이터 | 구분 기준 (명문화) |
|------|-------:|-------------------|
| **추락** (792) | **사람이 높은 곳에서 떨어짐** | 동의어: 떨어짐·낙상. 고소작업·개구부·비계. |
| **낙하** (731) | **물체가 위에서 아래로 떨어짐** (수직) | 인양물·자재 낙하. 줄걸이 불량. |
| **비래** (143) | **물체가 튀거나 날아옴** (비수직·비산) | 불티 비산, 파편, 자재 튐. |

> 판정축: **사람이 떨어지면 추락 / 물체가 수직 낙하면 낙하 / 물체가 날아오면 비래**. 동일 작업에 추락+낙하 복수 공존 빈번(타워크레인 등) → coverage 평가에서 둘 다 must_include 가능.

#### (D) 협착 / 말림 / 끼임 계열

|------|-------:|-------------------|
| **협착** (522) | 두 물체 사이에 **끼임/눌림** | 동의어: 끼임·눌림. 회전체·중장비·구조물 사이. |
| **말림** (3) | 회전체에 **감겨 말려들어감** | 동의어: 감김. 컨베이어·드릴. 표본 극소. |

> 말림(3)은 표본 극소 → LLM이 협착으로 통합 분류해도 부분정답 허용. 단 입력에 "감김/말림/회전체에 감김" 명시어 있으면 말림 우선.

#### (E) 질환 / 근골격계 / 직업성 질환 — 건강 계열
| 형태 | 데이터 | 구분 기준 (명문화) |
|------|-------:|-------------------|
| **질환** (190) | 분진·흄·소음 등 **급·만성 건강장해 일반** | 용접흄·분진·피부질환. 광의 건강. |
| **근골격계** (124) | 중량물·반복동작에 의한 **근골격계 부담** | 인력운반·무리한 동작. |

### 2.2 LLM 분류 시 혼동 위험 쌍 목록 (eval-engineer·rag-architect 송신)

평가·프롬프트 few-shot에서 우선 대비할 confusable pairs (위험도 순):

| 우선 | 혼동 쌍 | 혼동 방향 | 완화 |
|:---:|--------|----------|------|
| ★★★ | 전도 ↔ 붕괴 ↔ 도괴 | 굴착·가시설에서 "넘어짐"과 "무너짐" 오인 | few-shot에 §2.1(B) 판정 트리 삽입. eval에서 이 3종 혼동을 별도 카운트 |
| ★★★ | 추락 ↔ 낙하 | "사람 떨어짐"과 "물체 떨어짐" 오인 (특히 복합 작업) | 주체(사람/물체) 명시 강제. 복합 시 둘 다 인정 |
| ★★ | 낙하 ↔ 비래 | 수직 낙하 vs 비산 | "튐/날아옴/비산" 어휘 신호 |
| ★★ | 베임 ↔ 찔림 ↔ 절단 | 상해 기전 오인 | 표본 극소 절단·찔림 과대분류 주의 |
| ★ | 협착 ↔ 말림 | 끼임 vs 감김 | 말림 표본 극소, 협착 통합 부분허용 |
| ★ | 질환 ↔ 근골격계 ↔ 질식 | 건강 vs 산소결핍 | 질식 분리, 분진/반복 신호 |

**동의어 정규화 가이드 (data-engineer 송신, SKILL 동의어 규칙 확장):**
- 낙상·떨어짐(사람) → **추락**
- 넘어짐·미끄러짐·뒤집힘 → **전도**
- 끼임·눌림 → **협착** / 감김·말려듦 → **말림**
- 충격·부딪힘 → **충돌**
- 비산·튐·날아옴 → **비래**
- 무너짐(지반/구조/동바리) → **붕괴** / 무너짐(가설/적치 국부) → **도괴**
- 전기충격·감전사 → **감전**

> [도메인 판단 필요]: '무너짐'은 단독으로 도괴/붕괴 양쪽 가능. 정규화 시 자동 치환하지 말고 대상(지반·동바리=붕괴 / 가설·적치=도괴) 문맥으로 판정. 모호하면 붕괴 우선(보수적, 중대성 높음).

## 3. 트리플 중복 7행 처리 권고

data_profile §10이 식별한 (중공종, 세부항목, 위험요인) 트리플 중복 7그룹. 실측으로 각 그룹의 강도·빈도·등급·중점등록을 확인했다(아래). **개선대책·코드값이 상이하므로 병합 금지가 원칙이며, 그룹별로 차등 권고**한다.

| 그룹 | 행번호 | 중공종 / 세부항목 | 재해형태 | 강도·빈도·등급·중점 (행별) | 처리 권고 |

### 처리 원칙 (data-engineer 송신 — clean.py §E13 보강)

1. **코드값(강도·빈도·등급·중점)이 하나라도 다른 그룹 → 유지(비파괴)**. 동일 위험요인 텍스트라도 평가 맥락이 달라 별개 레코드. G1·G3·G4·G5가 해당. **단순 dedup 금지** 재확인.
2. **코드값 완전 동일 + 개선대책 동일 → 1행 유지**, 잉여 행은 `dup_of=<keep_row>` 메타 기록(재발추적). 완전중복(§2, 0행)과 동일 처리.
3. **코드값 동일 + 개선대책 상이 → 병합 후보**. 개선대책을 `·`로 결합하여 단일 청크화(검색 recall↑). G2가 해당 가능(개선대책 대조 필요).
4. **전 그룹 비파괴 보존**: 원본 행번호는 메타에 유지. 어떤 경우에도 도메인 검토 없이 자동 삭제하지 않는다.

**결론**: 트리플 중복 7행 중 **실제 잉여(제거/병합 후보)는 코드값 완전 동일 그룹(G2 등)뿐**이며, 대부분은 강도·빈도·등급이 달라 **정당한 별개 위험 레코드**다. data_profile의 "단순 제거 금지" 권고를 도메인 차원에서 확정 지지한다.

## 4. 요약 — 타 에이전트 송신 액션

- **data-engineer**: ① '재해 사례' 133행 `is_classification_candidate=false`+`use_as_corpus=true` ② '공통 일반'·'기타공사' `classification_priority=low` ③ 트리플 중복 7행 §3 차등 처리(대부분 유지) ④ 동의어 정규화 §2.2 적용('무너짐'은 문맥 판정).
- **rag-architect**: 분류 모드에서 `is_classification_candidate=false` 배제, 근거/few-shot 모드에서는 포함. 혼동 쌍 §2.2를 few-shot 판정 트리로.
- **eval-engineer**: 혼동 쌍 §2.2를 오분류 에러 카테고리로 분리 집계. 전도↔붕괴↔도괴, 추락↔낙하를 최우선.
