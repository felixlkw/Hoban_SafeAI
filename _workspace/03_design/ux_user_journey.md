# UX User Journey — 호반 JHA Agent (Phase 3 Design)

> 작성: frontend-engineer · 기반: `jha-frontend-ux` SKILL + `jha_generation_template.md`(응답 스키마) + `safety_scope.md`(데모 5건/경계셀/갭).
> 원칙: 한 화면 한 결정 · AI 추천은 제안일 뿐(항상 수정/거절) · 인용 1클릭 · 경계셀 자동확정 금지 · 갭은 refuse.

---

## 0. 핵심 result_type → UX 거동 매핑 (전 여정 공통)

| `result_type` / flag | UX 거동 | ERP 등록 |
|----------------------|---------|:--------:|
| `ok` | 정상 카드(분류·위험요인·대책·인용) | 검토 완료 후 가능 |
| `low_confidence` | 상단 노란 경고 배너 + 후보 2~3건 인라인 | 안전관리자 검토 필수 |
| `no_match` | empty state + "표준 데이터 없음 — 안전관리자 문의" CTA | 차단 |
| `refuse` (full: 석면·화학물질) | 거절 카드 + 조문만 표시 + "담당자 문의" CTA, 대책 없음 | 차단 |
| `data_gap=true` (partial: 밀폐공간) | 응답 가능 위험(추락)은 채움 + 갭 위험(질식)은 warnings 분리 + "대책 생성 안 됨" 명시 | 안전관리자 확정 후 |
| `boundary_cell=true` (곱16 s4×f4) | "상(잠정)"·"O(잠정)" 배지 + "안전관리자 판단 필요" + 확정 전 ERP 비활성 | **확정(상/중 택1) 전 차단** |

## 1. Persona별 여정

### 1.1 작업자 (worker) — 입력·제출 (모바일 우선)

```
[로그인 SSO]
   ↓
[① 자연어 입력]  "5층 옥상에서 타워크레인 분해·해체 작업"
   - placeholder 로테이션 / 음성 입력 / 자동완성 / 30~150자 가이드 / 최근 5건
   ↓ (제출)
[로딩]  스켈레톤 + "AI가 작업을 분석하고 있습니다…"
   ↓
[② 분류 추천 확인]  대공종/중공종/세부항목 + confidence 바
   - "수정" 클릭 → 인라인 드롭다운(대안 2~3건). AI 결정 아님을 문구로 강조.
   ↓ (확인)
[③ 위험요인·대책 확인]  재해형태별 카드 + 등급 배지 + 대책 체크리스트
   - 인용 [R00042] 탭 → 모달(원문). 경계셀이면 "상(잠정)" 배지 노출.
   ↓ (제출)
[④ 안전관리자 검토 요청]  "검토 요청 완료. 결과는 알림으로 안내됩니다."
```
- 작업자는 **등급 확정·ERP 등록 권한 없음**(RoleGate). 경계셀·refuse는 "안전관리자 확인 필요"로만 표시.

### 1.2 안전관리자 (safety_manager) — 검토·확정 (데스크탑/태블릿)

```
[검토 대기 목록]  (제출자·작업명·등급·경계셀/갭 뱃지로 우선순위 시각화)
   ↓ (열람)
[제출 JHA 열람]  분류 + 위험요인 매트릭스 + 5×5 시각화
   ↓
[인용 원문 확인]  [R00042] 클릭 → 우측 사이드 패널(원본 행 하이라이트 + score)
   ↓
[등급·대책 최종 확정]
   - 경계셀: "상(잠정)/중" 라디오 선택 강제 → 사유 입력 → 확정
   - 일반: 검토 체크 → 미검토 항목 있으면 "확정" 비활성
   ↓ (서명: PIN 또는 SSO 재인증)
[ERP 등록]  진행 상태(대기/성공/실패) 표시
```

### 1.3 관리자 (admin) — 통계·이력

```
[대시보드]  제출/확정/등록 건수 · 등급 분포(상/중/하) · 경계셀 확정 비율
   ↓
[평가 결과 모니터링]  분류 정확도·인용 precision·refuse 적정성(eval 연동)
   ↓
[사용자 피드백 열람]  수정·거절 로그 → 모델 튜닝 입력
```

## 2. 경계셀(곱16) 검토 여정 — 안전관리자 전용 상세

```
[매트릭스에서 경계셀 카드 식별]
   배지: ⚠ "상(잠정)" + "O(잠정)" + "경계 셀(강도4×빈도4) — 안전관리자 판단 필요"
   ↓
[판단 근거 확인]  강도4×빈도4 위치(5×5 마커) + 통제 가능성 메모 입력란
   ↓
[등급 확정 라디오]  ○ 상(중점등록 O 유지)   ○ 중(중점등록 X로 강등)
   - 미선택 시: "확정" 및 "ERP 등록" 버튼 비활성 + 안내 텍스트
   ↓
[확정 사유 필수 입력]  (강등 시 사유 텍스트 필수 — eval 부분점수/감사 추적)
   ↓
[확정 → 잠정 배지 제거]  "상" 또는 "중" 확정 배지로 전환 → ERP 등록 활성
```
- 데이터 근거: 곱16 = 249상/35중 혼합, 강등 35행 전부 중점등록 X. UI는 **자동 확정 금지**, 사람 판단 강제.

## 3. 데모 5건 화면 흐름 (요약)

| # | 입력 | 기대 result_type | 핵심 UX 분기 |
|---|------|------------------|-------------|
| 1 타워크레인 해체 | "5층 옥상 타워크레인(T형) 분해·해체" | `ok` (상 후보, 경계셀 가능) | 추락·낙하·협착 카드 / 등급 상 배지 / §43 인용 / 곱16이면 경계셀 배지 |
| 2 굴착·흙막이 | "지하 2단 흙막이 가시설 후 터파기 굴착" | `ok`/`low_confidence` | 붕괴·전도 카드 / 중공종 후보 다수 → 분류 "수정" 드롭다운 부각 / §338~347 인용 |
| 3 거푸집·동바리 | "보·슬래브 거푸집 및 동바리 설치" | `ok` | 낙하·붕괴 카드 / 등급 중~상 / 건축 골조 coverage |
| 4 고소 용접·볼트 | "강교 벤트 위 H-Beam 용접·볼트 체결" | `ok` (복합) | 추락+감전+화재 다중 재해형태 탭 / 다중 인용(§43·44 + §301~321) / 감전 과대평가 주의 |
| 5 밀폐공간·맨홀 | "E/V PIT 내부 또는 맨홀 내 청소·점검" | `data_gap` (partial) 또는 `refuse` | 추락은 인용 채움 + 질식은 warnings("밀폐공간 절차 데이터 부족, 대책 생성 안 됨") + 중대재해 사례 인용 + 안전관리자 문의 CTA |

> 데모 동선: 1(간판 정상) → 2(분류 결정성) → 4(복합·다중인용) → 곱16 경계셀 시연 → 5(갭·refuse 가드레일). 모바일은 1·5를 우선 시연(작업자 흐름).
