## 12. 정제 단계 권고 사항 (Phase 2 적용)

아래 규칙을 `scripts/etl/clean.py`로 자동화한다. 수동 편집 금지, 원본 보존.

### A. 텍스트 정규화 (위험요인·개선대책)
1. **선행 불릿 제거** — `^\s*[-·•▪]\s*` 패턴 제거 (대상 7,585 셀). 불릿 유무는 의미 없음.
2. **다중 공백 단일화** — `  +` → ` ` (대상 3,004 셀).
3. **셀 내 복수 항목 분리** — ` - ` 구분자로 분할 후 `·`(중점) 또는 `\n`으로 정규화. 한 셀에 여러 개선대책/위험요인이 묶인 경우 청크 텍스트에서 가독성·검색 recall 확보 (rag-architect 합의 필요).
4. **비표준 공백 치환** — NBSP(U+00A0)/전각공백(U+3000) → 일반 공백 (대상 0 셀).
5. **양끝 trim** — 모든 텍스트 컬럼 strip.

### B. 분류 정규화
6. **분류 ID 부여** — 대공종/중공종/세부항목 정규명 → 안정 ID (`MJ###`/`SB###`/`DT####`). taxonomy_lookup CSV 3종 생성.
7. **표기 통일** — 영문 약어/한자 정규형 사전 적용 (safety-domain-expert 동의어 가이드 연동). 예: `T/C`·`T형`·`Tower Crane` → `타워크레인(T형)`.

### C. 코드값 검증·보정
8. **강도/빈도** — 비정수 0건 / 범위위반 0건. → float 정수화·범위 클램프 금지, 위반 행은 `data_quarantine.jsonl` 격리 후 safety-domain-expert 보고.
9. **위험등급** — 비표준 0건은 `[도메인 검증 필요]` 태그 (임의 보정 금지).
10. **중점등록** — 비표준 0건 동일 처리. 'O'/'X' 외 값은 격리.

### D. 정합성 처리
11. **등급 정합성** — 다수결 룰(하→중 임계곱 10, 중→상 임계곱 16) 기준 모순 35행에 `risk_grade_check=true` 메타 플래그 부여 + safety-domain-expert에게 원본값 유지/보정 판단 요청 (KRAS 매트릭스 실제 경계와 대조).

### E. 중복 처리
12. **완전중복** — 0행은 첫 행만 유지, 중복 chunk_id를 메타에 기록(재발추적).
13. **트리플중복** — 7행은 개선대책 상이 가능 → 자동 제거 금지. 개선대책까지 동일하면 §12와 합류, 다르면 병합(개선대책 `·` 결합) 후보로 safety-domain-expert 검토.

### F. 결측 처리
14. **결측 없음** — 모든 데이터 컬럼 결측 0. 결측 처리 불필요.

### G. 인코딩
15. **인코딩 정상** — U+FFFD 치환문자 0건. openpyxl(UTF-8) 정상 디코딩 확인. 추가 변환 불필요.
